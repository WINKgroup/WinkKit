WKRouter

public protocol WKRouter: URLRequestConvertible

A router is responsible to create the http request based on url, parameters, encoding… The created request should be used by a WKService. In your application module, create an extension to override the baseURL and headers attributes. A typical implementation of a router with enums, for a Post object in a social network app could be:

enum PostRouter: WKRouter {
    // The route to get comments of post by the id of the post
    case getComments(postId: String)

    var method: HTTPMethod {
        switch self {
            case .getComments:
                return .get
        }
    }

    var path: String {
        switch self {
            case .getComments(let postId):
                return "posts/\(postId)/comments"
        }
    }

    var params: [String: Any]? {
        switch self {
            case .getComments:
                return nil
        }
    }

    var encoding: ParameterEncoding {
        switch self {
            case .getComments:
                return JSONEncoding.default
        }
    }

}

  • headers Default implementation

    Headers associated to the request. By default this value is equals to defaultHeaders var. You can re-implement this var to add other header like Authorization or Accept-Language. In your application module, you can create a WKRouter extension and do something like:

    extension WKRouter {
    var headers: HTTPHeaders {
        var headers = defaultHeaders // get the default headers first
        headers["Accept-Language"] = "it" // add the accept language header with it value
        return headers
    }
}

  • baseURL Extension method

    The base url of back-end, something like https://www.backend.com/api/. In your application module, you can create a WKRouter extension and do something like:

    extension WKRouter {
    var baseURL: URL {
        return URL(string: "https://www.mybackend.com/api/v2/")!
    }
}
