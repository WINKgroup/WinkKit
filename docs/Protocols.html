Protocols

The following protocols are available globally.

  • WKBaseViewController

    All WKViewController, WKTableViewController, WKCollectionViewController conform to this protocol to achieve the MVVM feature and static instantiation feature.

    See more

    public protocol WKBaseViewController: class

    The base protocol Error of the web requests of the Framework.

    Declaration

    Swift

    public protocol WKError: Error
  • WKPropertiesInspectable

    Define which properties will be inspectable in the custom WKView. If you add a new property in this protocol, coformed classes will add the property in the UIView extension exactly like the following example:

    @IBInspectable open var theNewVar: CGFloat {
    get {
        // reutrn the value
    }
    set {
        // set the new value
    }
}

    Important

    You must add the annotation @IBInspectable to show the control in the InterfaceBuilder!
    See more

    Declaration

    Swift

    public protocol WKPropertiesInspectable: class

  • WKViewControllerViewModel

    The base WKViewModel that is already bound with ViewController lifecycle. Al methods have default implementation that does nothing, so you can implement only methods you want.

    See more

  • WKViewModel

    WinkKit is based on MVVM pattern. MVVM Wikipedia.

    A good intro guide for this pattern is iOS MVVM.

    Every WKViewController, WKTableViewController, WKCollectionViewController needs a WKViewModel to properly works, because in MVVM the View Controller acts as a simple View.
